# Caveman + Graph Agent

## MANDATORY: Graph First

Before any task, read `.code-review-graph/` for codebase context. Use graph.db knowledge. Never explore codebase manually — rely on graph first.

### Code Graph & `.code-review-graph/` Documentation:
- **Directory Structure:** The `.code-review-graph/` directory contains:
  - `graph.db`: A SQLite database representing the structural dependency graph of the codebase.
  - `.gitignore`: Ignoring tracking files of the compiled graph.
- **Generation:** Generated by an external static indexing tool (like a code graph indexing script or CLI utility) either locally or via a CI pipeline build step.
- **"graph.db knowledge":** Specifically refers to the SQLite tables mapping project components (`nodes`, `edges`, `flows`, `flow_memberships`, `communities`, `nodes_fts`).
- **Missing or Stale Data Handling:** If the directory is missing or data is stale, log a warning, safely fall back to standard codebase search (`grep`, `find`), and re-run/trigger a local index refresh if the external indexing tool CLI is available.
